= USAGE GUIDE

Above is example of top level heading. This is referenced for navigation (previous/next) of the pages in rendered training content.

Use this page (USAGEGUIDE.adoc source file) as reference of the syntax for content drafting.

On published HTML pages, an *estimated time* appears below the lesson title automatically.
It combines reading time for the page text (at 200 words per minute) with the duration of any embedded audio, video, and images or diagrams.
No author markup is required for local media in `modules/chapterN/images/`.

Each `image::` macro adds *15 seconds* of viewing time by default. Override with `role=media-seconds-N`, or set `role=media-seconds-0` to exclude decorative images:

Refer chapter1 > images.adoc for example.

[source,asciidoc]
----
image::architecture-diagram.png[]
image::complex-diagram.png[role=media-seconds-60]
image::red-hat-logo.png[role=media-seconds-0]
----

If the page includes any `audio::` macro, estimated time uses *whichever is longer*: full reading time for the page text, or the *combined* duration of all audio clips—not both. Video, image, and lab time are always added on top.

For hands-on lab or practice pages, set the expected completion time in minutes at the top of the page:

Refer chapter1 > section1.adoc for example.

[source,asciidoc]
----
= Lab Exercise: Deploy the Application
:page-lab-minutes: 30
----

For external audio, video files, or YouTube/Vimeo embeds where duration cannot be detected automatically, set the length in seconds:

Refer chapter1 > images.adoc for example.

[source,asciidoc]
----
audio::https://example.com/lecture.mp3[role=media-seconds-120]
video::VIDEO_ID[youtube,role=media-seconds-600]
----

You can also use the `end` attribute when you know the clip length: `audio::url[end=90]`.

For raw iframe embeds in passthrough blocks (`++++`), declare duration with `data-media-duration` on the `<iframe>` tag or a wrapper with class `videoblock media-seconds-N`:

Refer chapter1 > images.adoc for example.

[source,asciidoc]
----
++++
<iframe
  src="URL"
  data-media-duration="600"
  width="100%"
  height="600px"
  frameborder="0"
  allowfullscreen>
</iframe>
++++
----

See link:./modules/chapter1/pages/images.adoc[Chapter 1 — Images demo] for examples.

The home page (*modules/ROOT/pages/index.adoc*) shows the *total estimated duration* for the course (sum of all pages listed in navigation).
Run `npm run build` to refresh that total after you add or change content.

=== Collapsible blocks (Deep dive / Hint)

Use `[%collapsible]` on an *example* block so optional detail stays collapsed until the learner expands it. A title line starting with a dot (`.Title`) sets the clickable label (summary).

[source,asciidoc]
----
.Hint
[%collapsible]
====
Optional extra detail for curious readers.
====

.Deep dive: why this works
[%collapsible]
====
Longer explanation, lists, or source blocks go here.
====
----

To start expanded instead of collapsed, use `[%collapsible%open]` on the example block.

Use *NOTE* and *WARNING* for short callouts everyone should read. Use collapsible blocks for longer hints or deep dives that would clutter the main lesson path.

Collapsible behavior applies to the **HTML site** from Antora. PDF output from `asciidoctor-pdf` is not interactive; readers typically see all block content without expand/collapse.

Live examples: link:./modules/chapter1/pages/collapsible-demo.adoc[Collapsible blocks demo].

=== Adding an Arcade Interactive Demo to a QuickCourse

Refer chapter1 > images.adoc for example.

You can easily embed an Arcade interactive demo directly into your QuickCourse. Because the embed links directly to your Arcade project, any updates you make in the Arcade platform will automatically reflect in the course.

How to embed:

Copy the HTML snippet below.

Paste it into your QuickCourse where you want the demo to appear.

Replace the src URL ([https://demo.arcade.software/](https://demo.arcade.software/)...) with the shared link from your specific Arcade.

Edit the other settings (width, height, data-media-duration etc.) to match your requirements.
----
++++
<iframe
  src="https://demo.arcade.software/eUmqr0dRpwc8jSY2QVbH?embed&embed_mobile=inline&embed_desktop=inline&show_copy_link=true"
  width="100%"
  height="600px"
  data-media-duration="600"
  frameborder="0"
  allowfullscreen
  webkitallowfullscreen
  mozallowfullscreen
  allow="clipboard-write"
  muted>
</iframe>
++++
----
